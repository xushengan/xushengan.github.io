前端知识点概述

发表于 2018-08-20

前端知识点概述

关键词：node、前端

前言

工作需要，小运维要开始学习前端和后台的知识去搭建一个面向内部的系统，以前几次课程设计遇到需要前端的都没有做好，这次算是从头开始学习的。

为什么要用前端呢？

因为前端能方便操作，数据可视化之后能加直观。况且不通过界面，我暂时还不知道人们怎么和后台去交互信息。

前端是什么？由什么组成？

简单从百度百科查

前端即网站前台部分，运行在PC端，移动端等浏览器上展现给用户浏览的网页。随着互联网技术的发展，HTML5，CSS3，前端框架的应用，跨平台响应式网页设计能够适应各种屏幕分辨率，完美的动效设计，给用户带来极高的用户体验。

前端技术一般分为前端设计和前端开发，前端设计一般可以理解为网站的视觉设计，前端开发则是网站的前台代码实现，包括基本的HTML和CSS以及JavaScript/ajax，现在最新的高级版本HTML5、CSS3，以及SVG等。

——百度百科

所以前端有设计和实现，本文目前只讨论实现部分，html,css,js是最重要的。很多资料都说学习前端，主要就是搞定这三种语言，但是又没有讲清楚其各自的功能，下面简单对其介绍，详情可以自己再资料：

HTML

HTML是网页的骨架，HTML对内容的显示进行了标识，只要有html写好了，我们的网页也算有基本内容可以看了

CSS

只有html的页面，还是比较抽的，css可以为页面排版布局，让它更加漂亮。

JS

有了上面两个，页面已经比较好看了，但是html+css，只能做到页面之间的通过重新加载方式的方式转跳，如果想做出移动鼠标就动态显示标题的摘要信息，或者动态地从后台取数据填充html表格，都需要通过js帮忙。

怎么学习前端

既然前端就由这三个东西组成，那我就学习这三个东西就行了啊，为什么网上一搜前端，各种node,angular，vue,react，还有什么bootstrap，jQuery，elment-ui各种各种奇怪的东西，很多模板或者包都要npm下载，这又是什么鬼。下面浅谈对上面这几个东西的认识：

node.js

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.

我认为就是node就是一个承载JS语言的平台，它可以来做简单的后端，同时还有很多其他作用，比如包管理和作为一个server，我对其作为中间层的用法比较有感，前后端分离的话，如果全部js都在客户端再异步请求ajax，那么虽然服务器的压力小了，但是用户加载的速度就慢了，node本身可以解释js，那么就把js在node层解析完再回传HTML页面可能是一个较好的选择。

参考:node作为中间层的一些看法

vue.js、angularjs、react

这三个，我暂时的认识是JS框架，即可以让我们更简单地写JS，同时它们还有一些路由功能，代替java或python这种后端的路由层或者叫C层

Bootsrtap和elment-ui

目前的理解都是前端CSS框架，即有好看的样式，引入时候加class就可以直接使用。但是它们的js功能暂时还没有用到。不过还是有一些简单的js功能的。

jQuery

jQuery应该是对原生js的封装，使用jQuery写的依然是js功能，另外它对ajax的支持还是比较好的。

小结

目前的学习方向，由于组内的大家都没有使用vue等js框架的经验，因此我们选用bootstrap（css）和jQuery(js)作为前端的部分,有余力的话想去学习一下vue.js。上面是最近学习的一些体会，很多概念由于写作的时间原因没有去查官方的资料，以后有时间再整改。
