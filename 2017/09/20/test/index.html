1、基础权限

  • 权限（是相对于文件来说的）

    1.1、查看权限ls -l（ll）

    |rw-| r- |r-
    –|—|–
    拥有者的权限|所属组的权限|其他人

1.2、基本权限：

r readable，可读
文件 可以查看文件内容（cat,vim,more,less）
目录 可以查看目录
w writable，可写
文件 可以修改文件内容（vim）
目录 可以修改目录的内容（创建文件，目录，修改文件名，删除文件或目录）
x excutable，可执行
文件 可以修改文件内容（vim）
目录 可以切换到目录中（cd)
- 没有权限

1.3、进程访问模型：

  • 用户执行程序访问资源，进程在执行时，携带用户uid和gid和group附加组的id
  • 两者匹配值可获得相应的权限
  • id 查看当前用户
  • ll 查看文件权限

1.4、三种用户访问类型：

拥有者（owner）： 进程uid和资源的uid一致
所属组（Group）： 进程gid和资源的gid一致
其他人（oteher)： uid和gid都不匹配

umask 命令，查看用户的umask值

1.8、普通用户创建

1.8.1、文件权限：
默认/初始权限（666） rw- rw- rw-
-umask(root) — -w- -w-=644
rw- r– r–
-umask（普通用户） — — -w- =664
rw- rw- r–
1.8.2、目录权限：
初始权限（777) rwx rwx rwx
-umask(root) — -w- -w-=755
-umask（普通用户） — — -w- =775
  • 若普通用户umask改成333（-wx -wx -wx）则算出444而不是333，因为本来就没有x权限（r– r– r–）

  • ：在原有参数设定基础上，追加参数。
  • ：在原有参数设定基础上，移除参数。
    = ：更新为指定参数设定。
    A：文件或目录的 atime (access time)不可被修改(modified), 可以有效预防例如手提电脑磁盘I/O错误的发生。
    S：硬盘I/O同步选项，功能类似sync。
    a：即append，设定该参数后，只能向文件中添加数据，而不能删除，多用于服务器日志文件安全，只有root才能设定这个属性。
    c：即compresse，设定文件是否经压缩后再存储。读取时需要经过自动解压操作。
    d：即no dump，设定文件不能成为dump程序的备份目标。
    i：设定文件不能被删除、改名、设定链接关系，同时不能写入或新增内容。i参数对于文件 系统的安全设置有很大帮助。
    j：即journal，设定此参数使得当通过mount参数：data=ordered 或者 data=writeback 挂 载的文件系统，文件在写入时会先被记录(在journal中)。如果filesystem被设定参数为 data=journal，则该参数自动失效。
    s：保密性地删除文件或目录，即硬盘空间被全部收回。
    u：与s相反，当设定为u时，数据内容其实还存在磁盘中，可以用于undeletion。
    各参数选项中常用到的是a和i。a选项强制只可添加不可删除，多用于日志系统的安全设定。而i是更为严格的安全设定，只有superuser (root) 或具有CAP_LINUX_IMMUTABLE处理能力（标识）的进程能够施加该选项。

1、用chattr命令防止系统中某个关键文件被修改：

chattr +i /etc/resolv.conf

然后用mv /etc/resolv.conf等命令操作于该文件，都是得到Operation not permitted 的结果。vim编辑该文件时会提示W10: Warning: Changing a readonly file错误。要想修改此文件就要把i属性去掉： chattr -i /etc/resolv.conf

lsattr /etc/resolv.conf

会显示如下属性
—-i——– /etc/resolv.conf

2、让某个文件只能往里面追加数据，但不能删除，适用于各种日志文件：

chattr +a /var/log/messages
