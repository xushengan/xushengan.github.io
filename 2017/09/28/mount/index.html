分区和挂载

-linux的硬件设备需要挂载到指定的目录才能使用

1、分区

fdisk

重读分区

1、重启
2、partx -a /dev/sda两次）

2、格式化（制作文件系统mkfs）

  • mkfs -t TYPE DEVICE
    mkfs -t ext4 /dev/sda5

3、挂载

  • 挂载：将设备与目录关联，访问目录就是访问设备
  • 相关目录/etc/fstab和/etc/mtab
    注意的几个点：
    1、根目录是最早挂载的
    2、其他mount目录必须已建立
    3、同一时间一个mount point只能挂载一次
    4、所有分区在同一时间只能挂载一次
    5、如果卸载，必须保证挂载目录无进程占用

1、创建挂载点

mkdir /mnt/disk1

2、挂载mount

mount命令选项：

3、写入fstab

每次开机自动挂载，写入/etc/fstab

  • blkid /dev/sda5 查看UULD

第一列是文件系统的标识label或者/dev/sda/UUID
第二列是挂载目录
第三列是文件系统
第四列是挂载选项
第五列能够被dump指令使用
第六列是否在开机过程中使用fsck检查文件系统是否完整0否1是，2早于1

  • 其实/etc/fstab只是配置文件，实际的挂载记录在/etc/mtab和/proc/mounts里面，/etc/fstab会实时写这两个文件
    blkid /dev/sda5 查看UULD

4、卸载

  • 将设备与目录断开
    umount MOUNT_POINT(挂载点的目录)

卸载时，目录不能被其他进程占用
查看哪个进程使用当前目录
lsof 目录
fuser 目录

4、查看分区表

df -Th
